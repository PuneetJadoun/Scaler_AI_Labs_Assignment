# Assignment Goal

Build a Python application that reads a Microsoft Word (.docx) document containing Personally Identifiable Information (PII), detects all supported PII entities, replaces them with realistic fake values while maintaining consistency, and generates a new redacted .docx document.

The application should also provide an evaluation methodology and generate evaluation metrics.

# Supported PII Types

The detector must support at minimum:

* Person Names
* Email Addresses
* Phone Numbers
* Company Names
* Physical Addresses
* Social Security Numbers (SSN)
* Credit Card Numbers
* Dates of Birth
* IP Addresses

The architecture must allow new detectors to be added easily.

Example:

Passport Number

Driving License

Aadhaar

PAN

Bank Account

etc.

Adding a new detector should require creating a new detector module instead of modifying existing logic.

---

# Detection Strategy

Use a hybrid approach.

## Regex

Use regex for structured PII.

Examples:

* Email
* Phone
* Credit Card
* SSN
* DOB
* IP Address

Reason:

These have well-defined patterns.

---

## Named Entity Recognition (NER)

Use spaCy (or another strong NLP library if justified) for:

* Person
* Organization
* Address (best effort)

Reason:

These entities have no fixed pattern.

---

# Replacement Strategy

Never delete PII.

Use Faker.

Design the detector as an extensible system.

Future detectors should be pluggable.
