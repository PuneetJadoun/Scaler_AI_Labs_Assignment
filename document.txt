=========================================================
OBJECTIVE
=========================================================

Design and implement the complete Document Module.

The module should ONLY be responsible for reading Microsoft Word (.docx) files and converting them into structured Python objects.

The module must NOT perform

- Regex
- spaCy
- Presidio
- PII Detection
- Faker
- Replacement
- Evaluation
- Reporting

The module should only know how to work with Word documents.

=========================================================
ABOUT THE INPUT DOCUMENT
=========================================================

The input document contains

- Normal paragraphs
- Tables
- Company names
- Addresses
- Phone numbers
- Email addresses
- Headings

Much of the important information exists inside tables.

Therefore the reader must extract BOTH paragraphs and tables.

A paragraph-only reader is not acceptable.

=========================================================
INPUT
=========================================================

The input document will always come from the project's

input/

directory.

reader.py should accept a Path object or file path.

Do not hardcode filenames.

Use pathlib.

=========================================================
ARCHITECTURE
=========================================================

Implement the following architecture.

models/

document.py

Represents the entire document.

It should contain the complete structured representation returned by reader.py.

---------------------------------------------------------

paragraph.py

Represents one paragraph.

Every paragraph should contain at minimum

- unique id
- text

---------------------------------------------------------

table.py

Represents one table.

A table should contain rows.

---------------------------------------------------------

cell.py

Represents table rows and cells.

Design a clean architecture for rows and cells.

Do not use nested dictionaries.

Use dataclasses.

=========================================================
READER
=========================================================

reader.py should expose ONE public function.

Example

read_document(path)

It should

1. Validate input.

2. Open DOCX using python-docx.

3. Read all paragraphs.

4. Read all tables.

5. Populate all dataclasses.

6. Return one Document object.

Hide every python-docx implementation detail from the rest of the application.

Future modules should never interact with python-docx directly.
